# Генеалогическое древо рода Дымковых — Sunburst диаграмма

## Структура проекта

```
family_tree/
├── original_foto_tree_parts/   # исходные фотографии (part1–8, final_tree)
├── venv/                       # Python-окружение
├── parse_trees.py              # OCR-парсинг изображений
├── build_sunburst.py           # построение диаграммы
├── output/
│   ├── family_data.csv         # ГЛАВНЫЙ ФАЙЛ ДАННЫХ (редактируйте здесь)
│   ├── final_sunburst.html     # результат — открыть в браузере
│   ├── raw_ocr.txt             # сырой OCR (если запускали parse_trees.py)
│   └── ocr_partN.txt           # OCR по каждому листу
└── README.md
```

---

## Быстрый старт

### 1. Активация окружения
```bash
cd ~/claude_projects/family_tree
source venv/bin/activate
```

### 2. Построить диаграмму из текущих данных
```bash
python build_sunburst.py
```
Открыть результат:
```bash
xdg-open output/final_sunburst.html
```

### 3. Запустить OCR (после установки Tesseract)
```bash
# Установка Tesseract (один раз):
sudo apt-get install tesseract-ocr tesseract-ocr-rus

# Запуск OCR:
python parse_trees.py
```
OCR сохраняет текст в `output/ocr_partN.txt` для ручной сверки.

## Как редактировать данные

Основной файл данных: **`output/family_data.csv`**

Откройте в LibreOffice Calc, Excel или текстовом редакторе.

### Колонки

| Колонка      | Описание                                           |
|--------------|----------------------------------------------------|
| `id`         | Уникальный идентификатор (латиница, без пробелов)  |
| `name`       | Полное имя персоны                                 |
| `parent_id`  | `id` родителя (пусто для корня)                    |
| `birth_year` | Год рождения (необязательно)                       |
| `death_year` | Год смерти (необязательно)                         |
| `gender`     | `M` — мужчина, `F` — женщина                       |
| `location`   | Место рождения/проживания                          |
| `source_part`| Источник: `part1`, `part2`, ...                    |
| `notes`      | Заметки, уточнения                                 |

### Правила мужской линии

- `gender=M` — персона включается в диаграмму
- `gender=F` — персона хранится в CSV для справки, но не отображается
- После правки запустите `python build_sunburst.py` повторно

### Пример — добавить нового человека
```csv
gen5_NewPerson,Дымков Иван Сергеевич,gen4_SeMiPah,2001,,M,Минск,manual,Добавлено вручную
```
