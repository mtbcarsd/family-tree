## 1. КОНТЕКСТ ПРОЕКТА

**Цель:** Интерактивная Sunburst-диаграмма генеалогического древа рода Дымковых.
**Корень:** Тимофеев (1800 г.р.) — самый старший известный предок.
**Смена фамилии:** Тимофеев → Дымков (через сына Терентия).
**Принцип:** строго мужская линия (патрилинейное дерево), женщины хранятся в CSV но не показываются в диаграмме.
**Источники данных:** 9 фотографий рукописного/нарисованного дерева в `original_foto_tree_parts/` (part1–part8 + final_tree.jpeg).

## 2. ЧТО УЖЕ РЕАЛИЗОВАНО (итог предыдущих сессий)

### Файловая структура

```
family_tree/
├── app.py                  — Streamlit-приложение (ОСНОВНОЙ ПРОДУКТ)
├── build_sunburst.py       — CLI: строит static HTML из CSV
├── parse_trees.py          — OCR-парсинг фото (результат плохой, не используется)
├── send_email.py           — отправка HTML на dymkous@gmail.com через Gmail SMTP
├── serve.py                — HTTP-сервер для раздачи HTML по LAN (порт 8080)
├── requirements.txt        — зависимости pip
├── README.md               — инструкции для пользователя
├── venv/                   — Python virtual env (активация: source venv/bin/activate)
├── original_foto_tree_parts/
│   └── part1.jpeg ... part8.jpeg, final_tree.jpeg
└── output/
    ├── family_data.csv     — ГЛАВНЫЙ ФАЙЛ ДАННЫХ (69 записей)
    ├── final_sunburst.html — статический HTML (последний сгенерированный)
    ├── raw_ocr.txt         — объединённый OCR (низкое качество)
    └── ocr_part1.txt … ocr_part8.txt — OCR по частям (низкое качество)
```

### Что умеет Streamlit-приложение (`app.py`)

Запуск: `source venv/bin/activate && streamlit run app.py`

- Вкладка **Sunburst**: интерактивная диаграмма с zoom-in по кликам
- Вкладка **Таблица**: поиск по любому полю, сортировка
- Вкладка **Статистика**: распределение по поколениям (bar chart), топ по потомкам, хронология рождений
- **Боковая панель фильтров**:
  - Мужская линия / все персоны
  - Поиск по имени (с автоматическим добавлением предков)
  - Глубина поколений (slider)
  - Ветка/предок (выпадающий список)
  - Год рождения (range slider)
  - Место проживания
- Кнопка **"Скачать HTML"** — выгрузка текущего вида

### CSV-структура (`output/family_data.csv`)

Колонки: `id, name, parent_id, birth_year, death_year, gender, location, source_part, notes`

- `gender=M` → отображается в Sunburst
- `gender=F` → хранится для справки
- `id` — латинский уникальный ключ (например: `gen3_MiPah`)

## 7. ВАЖНЫЕ ДОГОВОРЁННОСТИ

- **Данные** вносятся вручную в `output/family_data.csv` — OCR не годится
- **Женщины** хранятся в CSV (`gender=F`) но Sunburst строится только по мужской линии
- **ID-соглашение:** `genN_ИмяОтец` (латиница, без пробелов), N = поколение
- **source_part:** указывает, с какого фото взяты данные (для последующей проверки)
- **notes:** ставить `?` если данные ненадёжны
